# 实验报告批改 Skill

这是一个面向 Codex 的实验报告批改 skill，用于按实验手册、任务书、教师要求或评分细则批阅学生 Word 实验报告。它的核心原则是：人工审核为主，脚本只做证据提取和机械写入，不能让脚本自动决定分数、扣分项、评语或抄袭结论。

## 适用场景

- 批量批改学生实验报告。
- 按实验手册、任务书或报告中的实验内容要求重新评阅。
- 提取 Word 报告中的文字、截图数量、补交标记等证据。
- 辅助识别疑似雷同报告，并交由 Codex 人工复核。
- 将已经人工确定的成绩和评语写入 Word 批阅版。
- 将已经人工确定的成绩写入 Excel 成绩汇总表。
- 区分宽松评分、严格评分、补交报告、抄袭报告等情况。

## 安装方式

### 方式一：克隆到 Codex skills 目录

在 Windows PowerShell 中执行：

```powershell
git clone https://github.com/songmingjun3/experiment-report-grading-skill.git "$env:USERPROFILE\.codex\skills\experiment-report-grading"
```

安装后目录应类似：

```text
C:\Users\你的用户名\.codex\skills\experiment-report-grading\
  SKILL.md
  scripts\
    extract_report_evidence.py
    plagiarism_candidates.py
    write_manual_results.py
```

重启 Codex，或开启新的 Codex 对话后即可触发。

### 方式二：手动复制

将本仓库中的文件复制到：

```text
C:\Users\你的用户名\.codex\skills\experiment-report-grading\
```

确保 `SKILL.md` 位于该目录根部，而不是多套一层目录。

## Python 依赖

skill 自带的脚本需要 Python 环境，并依赖以下库：

```powershell
pip install python-docx openpyxl
```

也可以使用仓库自带的依赖文件安装：

```powershell
python -m pip install -r requirements.txt
```

在新环境中建议先检查依赖：

```powershell
python scripts/check_environment.py
```

如果后续项目需要对截图执行 OCR，可检查可选 OCR 环境：

```powershell
python scripts/check_environment.py --ocr
```

OCR 的 Python 依赖可以通过以下命令安装：

```powershell
python -m pip install -r requirements-ocr.txt
```

OCR 还需要系统中安装 Tesseract OCR 并加入 `PATH`。OCR 结果只能作为证据线索，最终是否扣分仍必须由 Codex 人工判断。

Codex 使用本 skill 时，如果发现依赖缺失，应先说明缺失项和安装命令，并在获得授权后再执行安装；不会在用户未授权时静默安装依赖。

## 推荐实验报告目录格式

建议一个课程项目使用一个根目录，实验材料和学生报告按实验编号分目录放置：

```text
课程实验报告\
  实验1\
    实验1手册.docx
    成绩表.xlsx
    20230517001-张三.docx
    20230517002-李四.docx
  实验2\
    实验2手册.docx
    20230517001-张三.docx
    20230517002-李四.docx
```

也支持常见中文编号形式，例如：

```text
第1次实验
第一次实验
实验一
实验01
experiment1
```

学生报告文件名建议包含学号和姓名：

```text
20230517001-张三.docx
20230517002_李四.docx
```

补交报告建议在文件名、目录名、成绩表备注或教师名单中显式标记：

```text
20230517003-王五-补交.docx
补交\20230517003-王五.docx
```

skill 只在存在明确“补交、迟交、逾期”等标记时按补交处理，不会仅凭文件修改时间推断补交。

## 不建议放入批改目录的文件

为减少误处理，建议将以下文件单独归档或命名清楚：

- 已批阅版、最终批阅版、重新批阅版。
- 历史过程文件、临时草稿、备份文件。
- 非本次实验的报告或手册。
- Word 临时锁文件，如 `~$xxx.docx`。

证据提取脚本默认会跳过手册、任务书、成绩表、最终批阅版、重新批阅版和 Word 临时锁文件。

## 批改原则

有效评分依据按优先级使用：

1. 教师明确评分细则或补充要求。
2. 学生报告中的“实验内容、实验要求、实验任务”。
3. 实验手册、任务书、指导书中的操作步骤和参考结果。
4. 学生报告正文、表格、截图、OCR 证据和上下文。

实验结果是否扣分，主要看报告是否满足本次实验内容要求。实验手册主要用于判断操作过程是否合理、截图和结果证据是否可信。不能使用手册和报告都没有要求的工具名、命令、数据集或历史经验作为扣分依据。

## 脚本边界

本项目包含三个辅助脚本。

### 环境检查

```powershell
python scripts/check_environment.py
python scripts/check_environment.py --ocr
```

用途：

- 检查 Python 版本。
- 检查必需依赖 `python-docx`、`openpyxl`。
- 可选检查 OCR 依赖 `Pillow`、`pytesseract` 和 Tesseract 可执行文件。
- 输出缺失依赖和建议安装命令。

限制：

- 不自动安装依赖。
- 不访问网络。
- 不修改项目文件。

### 证据提取

```powershell
python scripts/extract_report_evidence.py "课程实验报告目录" --out evidence.csv --skipped-out skipped.csv
```

用途：

- 批量定位 `.docx` 报告。
- 识别实验编号、学号、姓名、补交标记。
- 提取实验内容、实验过程、实验总结的文字摘录。
- 统计图片数量。
- 记录读取错误和跳过文件。

限制：

- 不判断实验是否完成。
- 不生成扣分项。
- 不生成最终成绩。
- 不生成最终评语。

### 疑似雷同候选

```powershell
python scripts/plagiarism_candidates.py evidence.csv --out plagiarism_candidates.csv
```

用途：

- 基于证据 CSV 生成同一实验内的文本相似候选。
- 输出“疑似雷同，待人工复核”线索。

限制：

- 不能直接判定抄袭。
- 不能直接给 0 分。
- 不能直接写入蓝色标记。

抄袭结论必须由 Codex 人工对比两份或多份报告的过程描述、实验结果证据、截图内容和实验总结后作出。若人工确认高度雷同，相关报告均按 0 分处理，并在成绩表中以蓝色标记。

### 机械写入

```powershell
python scripts/write_manual_results.py manual_decisions.csv --out-dir 归档材料 --gradebook 成绩表.xlsx --gradebook-out 最终成绩汇总表.xlsx
```

输入 CSV 必须包含：

```text
实验, 学号, 姓名, 源文件, 最终成绩, 最终评语, 人工判定依据
```

用途：

- 将已经人工确定的评语和成绩写入 Word 批阅版。
- 可选更新 Excel 成绩表。
- 补交报告可标红。
- 抄袭报告可标蓝。
- Word 结构异常时记录异常 CSV 并跳过该报告写入。

限制：

- 不计算成绩。
- 不生成评语。
- 不自动判断补交或抄袭。
- 不把评语追加到 Word 文末。
- 不清空教师签字、日期、备注等模板固定内容。

## 人工判定 CSV 示例

```csv
实验,学号,姓名,源文件,最终成绩,最终评语,人工判定依据,是否补交,标记
1,20230517001,张三,F:\课程实验报告\实验1\20230517001-张三.docx,86,"1. 实验结果：符合实验要求，结果较完善。2. 实验过程：步骤描述较完整。3. 分析与总结：总结较完整。4. 格式规范：格式规范，排版整齐。总评：报告基本完成实验要求，本次实验得分86分。","人工核对实验内容、手册步骤、截图证据和总结后确定。",否,
1,20230517002,李四,F:\课程实验报告\实验1\20230517002-李四.docx,78,"1. 实验结果：结果证据略有不足。2. 实验过程：个别关键步骤说明略简略。3. 分析与总结：总结略简短。4. 格式规范：轻微格式不一致。总评：报告基本完成实验要求，本次实验得分78分。","人工核对实验内容、手册步骤、截图证据和总结后确定。",是,补交
```

`人工判定依据` 不能为空，否则写入脚本会拒绝执行。

## 推荐工作流

1. 准备课程根目录、实验手册、任务书、学生报告和成绩表。
2. 让 Codex 使用本 skill，并明确宽松或严格评分标准。
3. 运行证据提取脚本，生成证据 CSV 和跳过文件清单。
4. Codex 逐份人工阅读实验内容要求、手册步骤、报告正文、截图和 OCR 线索。
5. Codex 人工形成扣分项、评语、成绩和人工判定依据。
6. 使用写入脚本生成 Word 最终批阅版和 Excel 成绩汇总表。
7. 检查异常清单，人工处理 Word 结构异常或无法匹配成绩表的报告。

## 输出文件建议

建议最终归档目录命名为：

```text
实验报告归档材料\
  实验1最终批阅版\
  实验2最终批阅版\
  最终成绩汇总表.xlsx
  最终人工评分明细.csv
  写入异常记录.csv
  评分细则说明.txt
```

`评分细则说明.txt` 应写成给教师、学生或教学归档人员阅读的评分规则说明，只包含评分依据、总分构成、扣分梯度、补交和抄袭处理、成绩等级等内容。不要在该文件中写 Codex 执行过程、脚本用途、OCR 线索、人工判定表生成方式或写入脚本说明。

## 重要注意事项

- 不能要求脚本全自动评分；脚本只能辅助。
- 格式是否规范必须由 Codex 人工查看版面判断，不能只看短行比例、图片数量或 XML 统计值。
- 截图不是简单分类，需要判断截图内容是否支持文字描述和实验要求。
- OCR 识别不完整时，不能直接判定缺失，应结合截图可见内容和上下文人工复核。
- 学生可见评语应使用自然的教师批阅口吻，不写入“Codex 人工审核、脚本提取、OCR 线索、证据索引”等内部流程或技术痕迹。
- Word 结构异常时，只记录异常并跳过写入，不要把评语追加到文末。
