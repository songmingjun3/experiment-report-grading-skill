# 实验报告批改

## 强制执行协议

执行任何实验报告批改、重新评阅、批量登记成绩或抄袭判定任务时，必须先遵守以下协议；这些协议优先级高于后续所有流程描述和脚本说明：

1. 必须在开始批改前明确声明本次采用“人工审核为主、脚本只辅助”的口径；如果用户要求全自动评分，必须说明本 skill 不支持由脚本自动决定分数。
2. 必须把“证据提取”和“评分判定”拆成两个阶段：脚本只能完成证据提取、OCR、截图联系表、文本摘录、格式线索提取、备份和机械写入；分数、扣分项、评语、抄袭结论和格式是否规范必须由 Codex 人工阅读材料后决定。
3. 必须为每份报告或每个批量评分分组保留人工判定依据，至少包括：实验编号、学号、姓名、实验结果扣分、实验过程扣分、分析总结扣分、格式扣分、最终分数、人工依据摘要。没有人工判定依据时，不得写入 Word 或 Excel。
4. 不得把脚本输出的相似度、关键词命中、截图数量、OCR 缺失、文本长度、图片分类、格式统计值、历史规则、模型推测直接转换为扣分或成绩；这些只能标记为“待人工复核线索”。
5. 如果报告数量过多导致当前轮无法逐份人工审核，必须分批处理或停止并向用户说明“尚未完成符合 skill 要求的人工审核”；不得为了完成批量任务而降级为脚本自动评分。
6. 批量写入前必须进行一次自检：确认最终分数来自人工判定表或人工审核结论，而不是脚本生成；确认补交红色标记和抄袭蓝色标记也来自人工确认。
7. 最终汇总必须写明“评分判定方式：Codex 人工审核；脚本用途：证据提取/机械写入”，并列出任何未完成人工复核的异常项。

## 最高优先级原则

所有实验的有效评分依据必须来自本次实验对应的实际材料，并按以下优先级使用：

1. 教师明确评分细则或补充要求优先。
2. 实验报告内的“实验内容/实验要求/实验任务”是实验结果是否扣分的主要验收口径；如果它比实验手册写得更多，结果评分应检查是否完全满足报告中的实验内容要求。
3. 实验手册、任务书、指导书主要作为完成实验内容的参考操作步骤和证据校验依据，可用于判断实验过程是否合理、截图是否支撑步骤和结果。
4. 学生报告中的文字、表格、截图、OCR 识别结果用于证明是否满足“实验内容要求”和参考手册步骤。
5. 不得使用实验报告内容要求、实验手册或教师要求中都没有出现的工具名、命令、步骤、数据集、算法或关键词作为扣分依据。
6. 不得因为某课程历史经验、脚本预置关键词、其他实验内容、常见软件栈或模型推测而扣分。
7. 如果脚本或历史规则与报告实验内容/实验手册冲突，以报告实验内容和教师要求为准；冲突项必须忽略并记录为“规则不适用”，不能写入学生评语。
8. 实验完整性、实验结果是否满足实验内容要求、实验过程是否合理、格式是否规范、扣分项、评语和最终成绩必须由 Codex 人工审核材料后判断；脚本不得代替人工进行这些推理。
9. 脚本输出只能作为待核验的证据素材或机械写入工具，不得把脚本给出的“缺失/满足/扣分/得分”直接作为最终批改结论。
10. 抄袭或高度雷同只能由 Codex 人工对比两份或多份实验报告的文本和截图后判定；相似度脚本、文本重复率或截图数量只能作为“疑似雷同”线索，不能直接定性。若人工确认两份同一实验报告在实验过程描述、实验结果证据和实验总结中均高度雷同，则两份报告均判定为抄袭，本次实验成绩均为 0 分，并在成绩汇总表中以蓝色标记得分。

示例：如果报告“实验内容”要求完成数据导入、连接、转换和结果验证，即使实验手册只是给出参考步骤，也应按这些实验内容检查最终结果；如果报告和手册都没有要求 Kettle、Spoon，则不能因为未出现这些术语而扣分。

## 通用工作流

1. 确认批改范围：定位课程根目录、实验目录、实验手册/任务书、成绩表和学生报告文件。
2. 确认评分标准：如果教师或用户已明确指定“宽松/严格/自定义细则”，按指定标准执行；如果未指定，必须先向用户提供“宽松标准”和“严格标准”两个选项并等待选择，不得静默使用历史默认值。批量写入前也必须确认本次口径。
3. 确认是否补交：只有文件名、目录名、成绩表备注、教师名单或报告内明确出现“补交/迟交/逾期提交”等标记时，才按补交处理；不要仅凭修改时间、创建时间或历史推测判定补交。
4. 读取实验材料：优先读取教师评分细则、报告中的实验内容要求、实验手册/任务书；材料不全时先按可见材料评分，并在汇总中说明依据来源。
5. 建立两类要求清单：人工从报告“实验内容/要求/任务”提取结果验收清单；人工从实验手册/任务书提取操作步骤和证据参考清单。
6. 读取学生报告：可用脚本提取实验内容、实验过程/步骤、实验结果/分析、总结、截图、OCR 文本及上下文，但提取结果必须人工复核。
7. 核验证据：人工查看报告文字、截图、OCR 和截图上下文，判断其是否共同满足实验内容要求；OCR 漏识别时以截图可见内容和上下文为准。
8. 复核疑似雷同：如果脚本、人工阅读或用户指出报告疑似相似，只能将其列为“待人工复核”；Codex 必须人工对比报告过程描述、实验结果证据、实验总结和截图内容后，才能判定是否抄袭。
9. 按扣分制评分：由 Codex 人工依据评分细则或默认框架给出实验结果、过程、分析总结、格式规范扣分；格式是否规范也必须人工查看版面后判断，不采用脚本自动推理出的扣分。若已人工确认抄袭，跳过常规扣分计算，直接按抄袭规则给 0 分。
10. 形成人工判定表：批量写入前必须先形成可追溯的人工判定表或人工审核明细，逐项记录每份报告的扣分和依据；该表可以由 Codex 手工整理并用脚本保存，但表内结论不得由脚本推理生成。
11. 写入评语和成绩：脚本只可按已经人工确定的评语和分数机械写入 Word 或 Excel；默认保留原文件，另存批阅版；更新 Excel 前先匹配学号、姓名和实验列。
12. 汇总异常：无法定位手册、无法识别报告结构、文件锁定、成绩表匹配失败、疑似雷同但未完成人工复核、报告未完成人工审核等情况要单独列出。

## 要求清单

结果验收清单优先来自学生报告中的“实验内容/实验要求/实验任务”。提取时只保留可评价、可验证的内容：

- 必须完成的实验任务或操作步骤。
- 必须出现的命令、配置、数据表、文件名、算法、工具界面或结果输出。
- 必须提交或展示的截图、表格、运行结果、查询结果、图表、模型结果、分析结论。
- 教师明确给出的评分细则或额外要求。

实验手册清单用于辅助判断过程和证据：

- 手册中的操作步骤可用于判断实验过程是否合理、顺序是否基本正确。
- 手册中的截图、命令、界面、输出示例可用于判断截图证据是否支撑报告描述。
- 如果报告实验内容要求比手册更多，结果扣分应按报告实验内容要求执行。
- 如果报告实验内容要求为空或明显只是模板占位，再回退到实验手册/任务书作为结果验收依据。

不要把以下内容作为独立扣分依据：

- 实验目的是否写得详细，除非教师明确要求评分。
- 实验背景、原理介绍是否完整，除非评分细则明确要求。
- 手册中没有出现的扩展工具、替代软件、常见关键词或脚本预置关键词。
- OCR 没识别出的内容，除非结合截图上下文能确认确实缺少或不一致。

当手册内容较长时，优先从“实验步骤”“操作步骤”“实验结果”“提交要求”“评分标准”等部分提取操作参考。不要从其他实验、文件夹名称或已有学生报告反推要求。

## 截图与 OCR 核验

截图不是简单按 `terminal/gui/blank` 分类后扣分，而是作为证据核验：

- 将截图与相邻文字、图题、步骤编号、报告实验内容要求和实验手册参考步骤对应起来。
- OCR 识别截图中的命令、表名、字段名、文件名、工具窗口标题、状态、查询结果、图表文字等证据。
- 判断截图是否能证明报告文字描述的操作或结果。
- 判断截图和文字是否共同满足报告实验内容要求；同时参考手册判断操作路径是否可信。
- 截图类型只作为辅助信息，不能作为最终扣分依据。
- 绿色对勾、运行成功、转换完成等状态证据只能证明作业或模块运行状态，不能替代最终结果证据。
- 对“保存到某表、查询前 N 条记录、验证结果正确性”等要求，必须能在截图/OCR或上下文中看到目标表名、查询语句、查询结果记录或等价结果证据；不能仅凭报告文字自述或绿色对勾判定完全满足。
- 上述规则要泛化到不同课程：凡是实验内容要求包含保存、查询、验证、输出、提交、展示、绘图、统计、训练、预测、分类、聚类、可视化等结果动作，都要从该要求文本中抽取对象和输出形态，并核验报告文字、截图/OCR或图表是否真的提供了对应证据。

扣分应谨慎：

- 截图能支撑报告实验内容要求中的核心结果：不扣分。
- 截图模糊、OCR 不充分但文字和上下文基本合理：按已选择评分标准轻扣，或在人工确认可见证据足够时不扣。
- 报告声称完成某结果，但截图明显无关、缺失，或与报告实验内容/手册参考步骤不符：按已选择评分标准中的“结果证据不足”或“缺少关键结果”扣分。
- 查询/保存/验证类结果缺少目标表、查询语句或查询结果证据：按已选择评分标准中的对应梯度扣分，不使用固定分值。
- 不要把某个实验的专有对象（如某张表、某个软件、某条命令）硬编码为跨课程规则；应从当前报告实验内容和实验手册中动态提取对象、动作和结果证据。
- 多个关键结果缺少证据时，可累计扣分，但累计值必须受该评分项总分上限约束，并与已选择的宽松/严格口径一致。

## 默认评分框架

如教师提供明确评分细则，优先使用教师细则。否则使用以下默认扣分框架：

- 实验结果与报告实验内容要求：40 分。
- 实验过程描述：20 分。
- 实验结果分析与总结：20 分。
- 报告格式规范：20 分。

### 梯度扣分原则

每个评分项都必须按问题程度给出梯度，不要只有“不扣/扣满”两档。运行批改前必须确认使用“宽松标准”还是“严格标准”：

- 宽松标准：适合过程性练习、初次批改、教学反馈导向场景。轻微问题扣 1-2 分，一般问题扣 3-4 分，明显问题扣 5-8 分，严重问题扣 9 分以上。
- 严格标准：适合终结性评分、用户要求加大扣分力度、需要明显拉开区分度的场景。轻微问题扣 2-4 分，一般问题扣 7-10 分，明显问题扣 12-15 分，严重问题扣 16 分以上。
- 自定义标准：若教师给出明确评分细则或用户临时指定区间，优先按该细则执行，并在汇总文件中记录口径来源。

同一评分项内多个小问题可以累计，但要避免重复扣同一原因。累计后仍应检查总扣分是否与报告整体完成度相称：能基本完成实验的报告不应因若干轻微问题被压得过低；核心结果无法证明的报告也不应只象征性扣分。

默认等级区间：90 分及以上为优秀，75-89 分为良好，60-74 分为合格，60 分以下为不合格。若用户给出不同区间或教师有明确细则，优先使用用户/教师要求。不要出现未归类分段。

汇总文件必须写明“评分标准：宽松/严格/自定义”。写入 Word 或 Excel 时，成绩和评语必须来自该已确认评分标准下的人工审核结论。

补交报告起评分规则：标记为补交的报告总分以 80 分为起评分，再按已选择的宽松/严格/自定义标准执行各项扣分，即 `最终成绩 = 80 - 实验结果扣分 - 实验过程扣分 - 分析与总结扣分 - 格式规范扣分`；非补交报告仍以 100 分为起评分。补交标记和起评分必须写入汇总文件，评语总评中可简要说明“按补交报告起评分 80 分计”。若教师另有补交扣分细则，优先使用教师细则。

### 抄袭/高度雷同判定

抄袭判定优先级高于常规扣分和补交起评分。一旦人工确认抄袭，本次实验成绩直接为 0 分，不再按补交 80 分或常规扣分框架计算。

判定条件必须同时满足：

- Codex 已人工打开并对比同一实验的两份或多份报告，不能只看相似度脚本输出。
- 两份报告的实验过程描述存在大段连续雷同，且不是仅由固定模板、实验手册原文、统一实验要求或通用命令造成。
- 两份报告的实验结果证据高度雷同，例如截图内容、截图顺序、图注、结果表述、查询/模型/聚类/输出结论等明显一致，缺少可区分的个人完成痕迹。
- 两份报告的实验总结或心得体会高度雷同，包含相同或近似的个人化表述、问题描述、改进意见、错别字、标点或语序。

不能直接判定为抄袭的情况：

- 仅实验目的、实验内容要求、模板固定文字或手册步骤相同。
- 仅使用同一命令、同一软件界面、同一数据集或同一算法。
- 仅文本相似度高，但截图、结果证据和总结存在明显差异。
- OCR 或相似度脚本给出高相似，但人工尚未完成报告文本和截图对比。

处理规则：

- 对人工确认抄袭的双方或多方，同一实验均记 0 分；不要只处罚其中一人。
- Word 评语应简洁说明“经人工对比，实验过程、实验结果和实验总结高度雷同，按抄袭处理，本次实验记 0 分”，不要展开过长证据链。
- 成绩汇总表中，人工确认抄袭的实验成绩单元格必须以蓝色标记；优先使用蓝色字体，也可使用浅蓝色填充。若同时存在补交标记，抄袭蓝色标记优先于补交红色标记。
- 汇总明细必须列出被判定抄袭的实验编号、学号、姓名、相互雷同对象、人工判定依据和最终 0 分结果。
- 如果只是疑似雷同但证据不足，不能给 0 分；应按常规评分，并在内部汇总或异常清单中标记“疑似雷同，待人工复核”。

### 实验结果

只根据“报告实验内容要求清单”判断；报告实验内容无有效要求时，才回退到实验手册/任务书。

- 完成主要实验内容要求，步骤结构和核心结果合理：不扣分。
- 结果证据有轻微不足，但截图、文字或上下文能基本证明完成：宽松扣 1-2 分，严格扣 2-4 分。
- 单个关键结果证据不足，例如保存目标表、查询前 N 条记录、验证正确性缺少明确截图或等价证据，但其他结果完整：宽松扣 3-4 分，严格扣 8 分左右。
- 缺少一个关键实验内容要求，或多个结果证据都较薄弱，但仍能看出主体任务已完成：宽松扣 5-7 分，严格扣 10-14 分。
- 缺少多个关键结果，或结果与实验内容要求明显不一致，主体完成度存疑：宽松扣 8-12 分，严格扣 15-22 分。
- 无法判断是否完成主要实验内容，或核心结果基本缺失：宽松扣 13 分以上，严格扣 23 分以上，视教师细则决定。

不得写“缺少某关键词”作为扣分原因，除非该关键词来自报告实验内容、手册或教师要求。

### 实验过程

过程评分关注是否能看出完成路径，并参考实验手册关键操作步骤判断是否合理；不要求逐条复述手册，但不能只用概括性图注替代主要操作过程。

- 步骤结构基本完整，能对应主要实验任务和手册参考路径：不扣分。
- 步骤基本完整，但个别参数、命令、配置或验证说明略简略：宽松扣 1 分，严格扣 2 分。
- 过程较简略，少量关键操作只写结果或图注，但仍能看出完整完成路径：宽松扣 2 分，严格扣 4 分。
- 过程存在一般省略，缺少一类关键操作说明，例如数据准备、连接配置、转换设计、运行验证中的一类：宽松扣 3 分，严格扣 7 分。
- 过程明显简略，多个关键操作只用界面截图或图题代替，参数配置、运行或验证说明不足：宽松扣 4 分，严格扣 9 分。
- 过程很简略，主要由“某界面/某页面/某结果”等截图说明组成，缺少具体操作过程，难以复现实验路径：宽松扣 5 分，严格扣 12 分。
- 过程严重缺失，只能看到少量结果或截图，基本无法判断如何完成实验：宽松扣 6-8 分，严格扣 15 分。
- 实验过程大面积缺失，或与实验内容明显不对应：宽松扣 9 分以上，严格扣 16 分以上。
- 手册关键步骤覆盖率也应泛化处理：从当前手册中动态提取带有下载、导入、连接、配置、测试、创建、运行、保存、查询、验证、输出、绘图、训练、预测等动作的步骤，不使用固定课程关键词。
- 明显整段复制手册且缺少个人完成痕迹：扣 5 分，严重雷同时按雷同规则处理。

### 分析与总结

- 有结果分析、问题说明或个人心得：不扣分。
- 总结存在但较简短，个人体会或问题说明略少：扣 1-2 分。
- 总结较空泛，主要是模板化表述，分析和个人见解不足：扣 3-4 分。
- 总结明显简略，只能看到少量收获描述，缺少结果分析或问题说明：扣 5-7 分。
- 基本缺少分析总结，或只有一句泛泛表述：扣 8-12 分。
- 完全缺少总结和分析：扣 13-20 分。

### 格式规范

格式规范必须由 Codex 人工查看报告版面后判断。脚本可以提取字体、行距、段落、图片尺寸、页面截图或版式线索，但不能直接判定格式扣分。

- 模板信息完整，实验内容和实验总结部分字体大小基本统一、行间距一致、缩进换行合理，整体文字和图片排版整齐：不扣分。
- 实验内容和实验总结部分存在轻微格式不一致，例如少量字号、行距或缩进不统一，但不影响阅读：扣 1-2 分。
- 存在一般格式问题，例如多处字号或行距不统一、部分图片与文字衔接不够整齐，但整体可读性较好：扣 3-4 分。
- 字体大小明显混乱、行间距多处不一致、缩进换行不合理，或图片与文字混排导致整体版面不整齐：扣 5-7 分。
- 缺少学号、姓名、实验编号等基本信息，或报告结构明显混乱：扣 5-8 分。
- 版面严重混乱、结构难以辨认，明显影响阅读和批改：扣 9 分以上。
- 不得仅因“短行较多”、XML 中存在多个字号、行距值不同、图片数量较多或脚本统计异常而扣格式分；这些只能作为人工观察排版是否异常的辅助线索，不能作为自动扣分标准。

## 评语要求

评语应简洁、完整。若某项有扣分，写扣分项摘要，不展开 OCR 明细、截图序号或过长证据链；若某项无扣分，不写“无明显扣分项”，而是对该项内容作合理正向概括：

有扣分时，评语和汇总中的扣分原因要体现梯度程度，例如“略有省略”“较简略”“明显简略”“严重缺失”，并与实际扣分值匹配；不要对不同程度的问题使用同一句评语。

```text
1. 实验结果：[符合实验要求，结果完善 / 结果证据略有不足 / 缺少某项关键结果]
2. 实验过程：[步骤描述详细，过程完整 / 个别说明略简略 / 过程较简略 / 过程明显简略 / 过程严重缺失]
3. 分析与总结：[总结较完整，有一定分析或心得 / 总结略简短 / 总结较空泛 / 总结明显简略 / 缺少分析和个人见解]
4. 格式规范：[格式规范，排版整齐 / 轻微格式不一致 / 多处格式不统一 / 图文排版明显不整齐 / 版面严重混乱]

总评：报告结构完整，基本达到实验要求，本次实验得到X分。
```

评语中的扣分项必须可追溯到手册要求或报告本身。不要写入“缺少 Kettle/Spoon”等手册外依据。

## Word 和成绩写入

- 默认保留原始报告，另存为批阅版或重新批阅版。
- 如果原批阅版被 Word 锁定，另存为 `（重新批阅版）`，不要强行覆盖。
- 重新评阅时，写入成绩前必须清理“成绩评定/得分”后已有数字，避免出现两个得分。
- 教师签字、日期、模板固定内容不要硬编码修改，除非用户明确要求。
- 更新 Excel 时优先按学号精确匹配，再按标准化学号、姓名匹配；找不到唯一匹配时记录异常，不要误写。
- 写入成绩汇总表时，若学生被人工确认为补交，必须将该学生对应实验成绩单元格标记为红色以区分；优先使用红色字体，若模板可读性更适合也可使用浅红色填充。若生成独立 `.xlsx` 汇总表，应同时标红补交学生的成绩单元格或整行；若只生成 CSV/TXT，则用“是否补交/起评分”字段文字标记，因为 CSV/TXT 不支持颜色。
- 写入成绩汇总表时，若学生被人工确认为抄袭，必须将该学生对应实验成绩写为 0 分，并将成绩单元格标记为蓝色以区分；优先使用蓝色字体，若模板可读性更适合也可使用浅蓝色填充。抄袭蓝色标记优先级高于补交红色标记。

## 脚本使用约束

本 skill 文件夹中的脚本只能作为工具实现，评分逻辑必须服从本文件规则。脚本不得执行或输出可直接作为最终结论的实验完整性推理：

- 脚本可以做：批量定位报告、读取 Word 表格文本、提取截图、生成截图联系表、执行 OCR、整理证据索引、提取字体/行距/段落/图片等版式线索、备份文件、按人工确定的评语和成绩写入 Word、按人工确定的成绩更新 Excel、收集已写入的成绩和评语。
- 脚本不可以做：判断实验是否完整、判断实验结果是否满足实验内容要求、判断实验过程是否覆盖关键步骤、判断总结是否合格、判断格式是否规范或应扣分、生成最终扣分项、生成最终评语、生成最终成绩。
- 脚本如果为了排序或提示输出“疑似缺失/疑似满足/候选扣分”，只能标记为“待人工复核”，不得写成确定结论，也不得自动写入学生评语或成绩。
- 脚本可以计算文本相似度、截图哈希、OCR 相似片段、图片顺序相似等线索，但只能输出“疑似雷同/待人工复核”；不得由脚本直接判定抄袭、直接给 0 分或直接写入蓝色标记。抄袭结论必须来自 Codex 人工对比报告文本和截图后的明确判断。
- 脚本不得根据文本长度、图片数量、关键词覆盖率、相似度、OCR 字段、截图分类、格式统计值或任何阈值自动生成最终扣分表；如果脚本生成了候选表，字段名必须使用“候选/线索/待复核”，不得使用“最终扣分/最终成绩/最终评语”。
- 机械写入脚本的输入必须是人工判定表；写入前应检查该表存在“人工依据摘要/人工判定依据/人工复核结论”等字段。缺少这些字段时，应停止写入。
- OCR 结果只能作为证据之一。OCR 未识别出表名、命令、结果记录或图中文字时，不得由脚本判定缺失；必须由 Codex 查看截图或上下文后再决定。
- 截图分类、绿色对勾、运行成功、日志完成等脚本识别结果只能作为辅助线索。是否足以证明实验结果，必须由 Codex 按实验内容要求人工判断。
- 字体大小、行距、缩进、换行、图片数量、图片尺寸、页面截图等格式线索只能辅助人工查看；不得由脚本根据阈值自动判定“格式不规范”或自动扣格式分。
- `regrade_all_with_ocr.py` 若继续使用，只能拆分为“证据提取/写入工具”；不得使用其中的自动评分、自动扣分、自动评语作为最终结果。
- `screenshot_analysis.py` 只能用于 OCR、图片提取、截图联系表和证据摘要；不得由其 `deduction`、`manual_coverage`、`missing`、`partial` 等字段直接决定分数。
- `collect_regrade_results.py` 只能用于收集、汇总已人工确认并写入的成绩或评语，不得反推新的成绩。

推荐执行顺序：

1. 用脚本批量提取每份报告文字、截图、OCR 和截图联系表。
2. Codex 逐份人工阅读实验内容要求、实验手册参考步骤和报告证据。
3. Codex 人工判断实验结果、实验过程、分析总结、格式规范是否扣分，并形成评语和成绩。
4. 用脚本把已经人工确定的评语、成绩写入 Word 或 Excel。
5. 用脚本生成汇总文件，但汇总中的扣分项和分数必须来自人工审核结论。

运行脚本前必须确认：

- 脚本没有把实验编号预置关键词、OCR 缺失、截图分类或历史规则作为扣分依据。
- 脚本没有自动把候选问题写入评语、成绩或最终汇总结论。
- 脚本没有自动把相似度或截图相似线索写成“抄袭”结论；所有抄袭 0 分和蓝色标记均来自人工确认。
- 脚本没有自动生成最终分数；所有最终分数均来自 Codex 已人工审核的判定表。
- 人工判定表或人工审核明细已存在，并且每份待写入报告都有对应的人工依据摘要。
- 手册要求和报告实验内容要求已由 Codex 人工确认。
- 输出评语和成绩来自人工审核结论，且没有包含手册外扣分项。
- 格式扣分由 Codex 人工查看字体大小、行间距、缩进换行和图文排版后决定，不使用“短行比例”、XML 统计值、图片数量或脚本阈值作为自动扣分依据。
- 修改前已备份需要写入的 Word 和 Excel 文件。

如果以上任一项无法确认，必须停止写入并向用户说明缺失的人工审核环节；不得继续生成最终成绩。

## 异常处理

- 找不到实验手册或任务书：暂停自动评分，向用户确认材料位置。
- 手册无法解析：列出可见文件并请求确认，不要凭经验评分。
- 报告结构无法识别：标记为“待人工复核”，不要强行给分。
- OCR 不可用：降级为文字和截图上下文核验，并在汇总中说明。
- OCR 结果与肉眼可见内容可能冲突：优先谨慎处理，必要时请用户复核截图。
